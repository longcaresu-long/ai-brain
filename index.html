01

系统定位与核心价值

数字化决策大脑 — 拟合超级个体

不是工具，是持续生长的 AI 决策伙伴

解决什么问题 方案上线才发现效果不好浪费研发成本；经验留在个人头脑里无法传承；决策缺乏数据支撑；团队能力提升没有系统化路径
核心价值 集合公司群体智慧，赋能每一个个体。越用越懂公司，越用越准。最终目标：提升平台收益，降低无效研发投入
与普通AI差异 普通AI没有公司上下文，给的是通用建议。这个系统持续积累公司认知，每次对话都基于你们公司的真实现状做判断，不是通用答案
AI角色定位 建议者，不是决策者。是否采纳取决于使用者。系统陪伴成长，会犯错，错误是校准机会，不是否定系统的理由
成长方式 陪伴式成长。每次真实决策后回测校准，置信度模型持续优化，知识库随使用越来越准

① 企业现状诊断

输入产品文档 → 评估效果预测和风险 → 输出建议推进/调整/不推进的结论。降低无效研发投入的核心机制。

持续追踪竞品版本更新记录，推断各阶段决策逻辑和问题解法，分析产品前中后期重点方向演变。

03

核心机制设计

多人格协作与群体智慧

不是仿真员工，而是提取群体的正向经验

人格来源 每个员工对应一个虚拟角色，提取其历史正向经验构建。员工离职后经验仍留存，成为公司资产
正向经验标准 方案考虑更全面 OR 代码效率更高，且被他人采纳验证 → 两者结合确认，系统自动判断
投票机制 触发多个虚拟人格各自提出方案，所有人格对所有方案投票（含自己的方案），按票数排序展示
结果展示 默认看综合最优解，需要时展开查看每个方案票数和理由，支持透明辩论过程查阅
员工成长 透明模式，员工自由选择开启。系统识别个人疏忽规律，在相关任务中针对性提醒
群体审查 员工主动提交文档/代码，系统调用全体人格集体检查，覆盖个人盲点，用群体智慧提升个体水平

职业智慧知识库 — 经验流动机制

个人经验 → 职业知识库 → 赋能全体 → 持续进化

员工方案/代码 → 系统提取正向经验 → 补充到职业知识库 → 自动推断适用条件 → 赋能其他员工
他人采纳/修改/放弃 → 修正适用条件 → 知识库持续进化
适用条件 系统从时间节点、排期、资源约束等参数自动推断每条经验的适用条件，随使用持续修正
跨职业抽象 通用思维模式可抽象赋能其他职业。例如"全局最优解"思路适用于所有涉及资源约束的决策场景
双轨输出 每个建议同时给出：当下可落地版本（考虑时间/人力/资源约束）+ 未来最终版本（理想状态完整方案）

权限控制与决策置信度

保障信息安全，建立决策信任

权限分层 引用内容可见，无权限则内容不可打开。可将对话分享给上级，由上级核实引用内容的真实性
置信度分级 高置信度（直接影响立项）· 中置信度（建议灰度验证）· 低置信度（明确说数据不足，建议先实验）
决策日志 每次评估结论和依据自动存档，人工覆盖系统结论时必须填写理由，形成可追溯的决策记录
隐私保护 数据过滤规则由系统执行，不开放给管理员配置。基于人权和隐私原则，只处理工作相关内容，不干预员工私人生活

05

推进路径（四阶段）

核心逻辑：由内而外 — 先让系统懂我们是谁和怎么思考，再懂我们的工作产出，再懂我们的产品，最后才向外看行业。系统有了公司视角后，给出的建议才是专属于你们处境的精准判断，而不是通用行业建议。
阶段 01 让系统懂我们 4~6 周

目标：建立公司认知底座和思维方式档案。系统在这一步形成"公司人格"，后续所有建议都通过这个视角过滤。这是整个系统的地基，最优先级。

1思想层定义 — 老大输入公司使命愿景价值观、责任博彩边界；老高输入社会层准则。整个系统的判断地基，优先级最高。
2组织架构输入 — 人员构成、职责边界、汇报关系、团队分工。建立系统对"这家公司有哪些人、各自负责什么"的基础认知。
3精华样本人工筛选 — 你和老板从历史数据中筛选20~30个最能代表公司思维方式的案例：最典型的成功决策、失败复盘、技术方案讨论。质量优先于数量。
4数据源接入 — Lark文档/群聊、禅道任务记录、代码仓库、复盘文档接入。系统开始自动摄入增量数据，隐私过滤规则同步启动。
5思维方式文档生成 — 系统基于精华样本 + 数据源，自动生成初稿认知文档。你和老板审阅确认后写入思想层。
6员工公示 — 透明告知全体员工系统读取哪些内容、过滤哪些内容、用于什么目的。建立信任基础。
最大风险：思想层定义如果迟迟等不到老大和老高的输入，整个系统启动会被阻塞。建议在本次汇报后给定具体输入截止时间。
↓ 系统已懂公司，开始理解工作产出
阶段 02 让系统懂我们的工作产出 6~8 周

目标：建立职业智慧知识库和多人格协作机制。系统开始理解每个人做了什么、怎么做的，提取正向经验沉淀为集体智慧。

1职业智慧知识库建立 — 按职业分类提取正向经验，自动推断适用条件，建立可跨职业抽象的通用经验层。
2虚拟人格初始化 — 基于阶段一数据为每个员工建立虚拟人格底色。个人数据不足时，用同职业群体经验填补，随使用持续修正。
3投票协作机制上线 — 多人格方案生成 + 投票排序 + 透明辩论过程查阅。员工可主动提交文档/代码请求群体审查。
4员工成长模块 — 透明模式，员工自由选择开启。系统识别个人疏忽规律，在相关任务中针对性提醒。
5权限体系建立 — 引用内容可见但受权限控制，对话可分享给上级核实，决策日志自动存档。
阶段二完成标志：系统能基于公司思维方式 + 员工历史经验，对一份新提交的方案文档给出有针对性的审查意见，而不是通用建议。
↓ 系统已懂人，开始理解产品
阶段 03 让系统懂我们的产品 8~12 周

目标：平台收益决策辅助系统全面上线。系统有了公司视角和团队认知后，再看产品数据，给出的建议才真正有价值。

1功能文档知识库 — 现有产品功能文档整理入向量知识库，AI 理解每个功能的设计意图和用户价值。
2数数科技接入 — 六模块指标体系配置上线（收益健康、用户留存、付费转化、游戏行为、渠道代理、活动效果）。
3用户画像体系 — 行为层 × 来源层 × 偏好层三层叠加，游戏场景状态变量接入，虚拟用户生成。
4AI推演引擎 — Supabase + Claude API 搭建，历史版本节点回测校准，三种使用模式上线（目标导向/方案预判/指标预测）。
5前端埋点补充 — 大厅入口点击、活动页停留、推送来源、游戏切换路径，补充前端行为路径数据。
阶段三完成标志：产品经理提交一份新活动方案文档，系统能结合公司思维方式 + 团队能力 + 历史数据，给出"建议推进/调整/不推进"的有依据结论。
↓ 系统已懂内部，开始向外看
阶段 04 对外延伸 持续迭代

目标：建立行业视野，发现差异化机会。有了前三阶段的公司认知底座，行业对标才能给出真正有针对性的差异化建议。

1竞品追踪情报库 — 重点竞品（5~10个，待补充名单）版本更新持续追踪，推断各阶段决策逻辑。
2版本考古分析 — 竞品 + 自身版本记录对比，分析产品前中后期重点方向演变，识别行业规律。
3差异化机会识别 — 基于行业对标 + 公司现状，主动发现你们能做而竞品没做的机会点，给出案例依据和理由。
4系统持续进化 — 每次真实决策后回测校准，置信度模型持续优化，知识库随使用越来越准。
数据来源：公开信息（应用商店、官网、新闻、用户评价）+ 行业报告 + 竞品功能变化追踪。不获取竞品私有运营数据，通过公开行为推断策略逻辑。

08

核心建议

汇报时的三个重点

预判老板可能的问题和你的应对

建议一：先做小，快速验证。
不要一开始就试图把整个超级个体系统做完。先把平台收益决策辅助系统做出来，证明"AI 能帮我们做出更好的研发决策"这件事是真实的。有了第一个成功案例，后续扩展的内部阻力会小很多，预算也更容易争取。
建议二：思想层必须在系统上线前定义好。
很多公司做 AI 系统，把思想层当成"以后再补充"的事情。但一旦系统上线、员工开始使用，再修改思想层会导致系统行为突然变化，破坏信任。思想层要在第一个模块上线前就写进去，哪怕是粗糙的版本也比没有强。
建议三：把"陪伴成长"写进团队共识。
系统早期会犯错，预测会有偏差。如果团队没有提前建立"系统需要时间成长"的心理预期，第一次重大失误就可能导致系统被放弃。建议在系统上线时向团队说清楚：前3个月是校准期，系统建议作为参考，不作为唯一依据，这段时间的所有预测误差都是在帮系统变得更准。

应对老板可能的质疑

vs 直接用ChatGPT 普通 AI 没有公司记忆，每次从零开始。这个系统持续积累你们公司的数据、经验、判断标准，给出的是专属于你们处境的建议，不是通用答案。
可信度问题 系统的每个结论都基于历史数据，不是黑盒。结论背后有数据依据，可以追溯，可以被质疑。即使预测偏差，可以解释"因为X所以预测Y，但实际是Z，说明Z这个变量之前没考虑到"——这是可接受的偏差。
如果系统判断错了 每次错误都会被记录和分析，成为系统下一次更准确的依据。系统不是买来即用的工具，是需要陪伴成长的伙伴，错误是成长的代价。
